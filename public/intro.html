help

Using gfxCardStatus improperly can negatively affect your battery life. This brief guide will give you some important tips to help avoid common problems.

i. os x mountain lion.

Far fewer apps will cause the more power-hungry graphics to turn on if you're using OS X Mountain Lion (10.8).

Even if you haven't installed Mountain Lion, I recommend staying on Dynamic Switching as much as possible. Don't switch to Integrated Only or Discrete Only unless you absolutely need to. See switching for more details, as well as some important information about power source-based switching.

ii. apps.

Certain apps will cause the more power-hungry discrete graphics to turn on when opened. Most trivial apps should no longer do this if you're running Mountain Lion, but there are plenty that still do. Among them are:

  • Google Chrome, Firefox
  • Skype
  • Parallels, VMware Fusion, VirtualBox
  • Apple Keynote

Virtualization apps like Parallels, VMware and VirtualBox all have valid reasons for using the more power-hungry graphics, as does Keynote. Google Chrome has had issues with graphics switching since more than two years ago, and they're still not fixed. Safari is both faster, and more graphics switching-friendly. I strongly recommend using Safari instead.

iii. external displays.

You cannot use external displays (including projectors) while using gfxCardStatus in Integrated Only mode. To use a display, stay on Dynamic Switching.
